# 現状とのズレ一覧

## 1. フロント payload（customer）

**現状:** 冗長なネスト（customer, delivery, estimate, consent）を含む。フラット項目も重複。
**対応:** 正本に必要なフラット項目のみ送信。GAS側で canonical に正規化。

## 2. フロント payload（driver）

**現状:** files のキーが「免許証（表）」「免許証（裏）」「車検証」「任意保険」の4つのみ。
**対応:** 必須7種＋任意5種の書類キーを追加。キーはGASのファイル名マッピングと一致させる。

## 3. Code.gs saveDriverFiles_

**現状:** fileBaseNames が古い（車両（前面）、自賠責、経営届出書、貨物保険など）。Appのキーと不一致。
**対応:** 正本の12種の書類マッピングに統一。

## 4. ドライバー登録シート列

**現状:** 書類提出状況の列がない。Drive保存先のみ。
**対応:** 各書類の提出有無列を追加（免許証表提出〜その他資料提出）。

## 5. SETUP.gs シート名

**現状:** KEEP_SHEET_NAMES に「ドライバー登録」→ ユーザー指定は「ドライバー登録」で一致。
**対応:** 設定シート名を「設定」に統一（現状「設定」でOK）。4シート以外削除は既に実装済み。

## 6. Setup.html

**現状:** 別プロジェクト（Drivers/Customers/PublicDrivers/Settings）の古いセットアップ。
**対応:** 軽貨物TAKEでは未使用。SETUP.gs が正本。Setup.html は削除または無視。

## 7. 通知文面の項目参照

**現状:** handleCustomer_ / handleDriver_ 内で payload から直接参照。正規化オブジェクト経由ではない。
**対応:** canonical オブジェクトを1つ作り、保存・メール・LINE・ユーザー確認メールはすべてそこから生成。

## 8. 稼働エリア列名

**現状:** DRIVERS_HEADERS で「稼働エリア」
**対応:** そのまま維持。
