# 実行手順（整合性整理後）

## 1. フロントビルド・確認
```bash
npm run build
npm run dev
```
- 配送相談: 見積依頼・正式依頼のpayloadが正本構造に準拠していること
- ドライバー登録: 必須7種＋任意5種の書類アップロードが表示されること

## 2. GAS反映
```bash
npm run gas:push
```

## 3. スプレッドシート環境リセット
1. スプレッドシートを開く
2. 拡張機能 → Apps Script を開く
3. メニュー「軽貨物TAKE」→「環境を強制リセット」を実行
4. 4タブ（配送依頼、ドライバー登録、ログ、設定）のみ残ることを確認
5. 配送依頼・ドライバー登録のヘッダー行が最新仕様になっていることを確認

## 4. Script Properties 確認
以下が設定済みであること:
- SPREADSHEET_ID
- ADMIN_EMAIL
- LINE_CHANNEL_ACCESS_TOKEN
- LINE_TO_USER_ID または LINE_TO_GROUP_ID
- DRIVE_FOLDER_ID（ドライバー登録の書類保存先）

## 5. 動作確認
- 配送相談フォームから送信 → 配送依頼シートに保存、管理者メール・LINE・ユーザー確認メール
- ドライバー登録フォームから送信（必須7種の書類を添付）→ ドライバー登録シートに保存、Driveにフォルダ作成、管理者メール・LINE・ユーザー確認メール

## 注意
- 既存の配送依頼・ドライバー登録データがある場合、forceResetTakeEnvironment はヘッダー行のみ上書き。データ行は残るが、列数が増えた場合は手動で調整が必要な場合あり
- 初回または列構成を完全に揃えたい場合は、バックアップ後にシートを削除してから forceResetTakeEnvironment を実行すると4シートが新規作成される
