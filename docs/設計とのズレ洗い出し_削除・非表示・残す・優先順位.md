## 2. 非表示にすべき要素

初期公開範囲は「トップ・配送相談フォーム・ドライバー登録フォーム」のみ。以下は**導線を消し、通常操作では表示されないようにする**。View コンポーネントや state は残してよい（将来の拡張用）。

### 2.1 ナビ・ルーティング

| 対象 | 現状 | 対応 |
|------|------|------|
| ViewState | `simulator` / `consult-vehicle` / `consult-vehicle-buy` / `consult-vehicle-repair` / `consult-vehicle-inspect` / `consult-business` / `takesan-log` が存在 | これらの view へ遷移する**ボタン・リンクのみ削除**。state と `currentView` の分岐は残すか、該当 view を描画しないようにする（URL 直叩き対策は後回しで可） |

### 2.2 トップページ

- ヒーローの主役が「協力ドライバー登録プール」で、ボタン順が「1. ドライバー登録 2. 運営者プロフィール」→ **文言・構成の変更**で対応（「削除」ではなく「修正」）。「非表示」対象ではない。

### 2.3 お客様向け（CustomerTopView）

| 箇所 | 内容 | 対応 |
|------|------|------|
| 1772–1774行目 | 「💰 運賃シミュレーター」ボタン→`setView('simulator')` | ボタン自体を非表示または削除 |
| 1780–1783行目 | 「🔧 車両トラブル相談」ボタン→`setView('consult-vehicle')` | 同上 |
| 残す | 「📦 配送・お見積り依頼」→`consult-delivery` | 配送相談のみ導線として残す |

### 2.4 ドライバー向け（DriverTopView）

| 箇所 | 内容 | 対応 |
|------|------|------|
| 1804–1807行目 | 「🏢 開業トータル相談」ボタン→`setView('consult-business')` | ボタン自体を非表示または削除 |
| 残す | 「📝 協力ドライバー登録」→`register` | 登録のみ導線として残す |

### 2.5 main 内の描画分岐

| 箇所 | 内容 | 対応 |
|------|------|------|
| 168–176行目 | `simulator` / `consult-vehicle` / `consult-vehicle-buy` / `consult-vehicle-repair` / `consult-vehicle-inspect` / `consult-business` / `takesan-log` の各 `currentView === '...'` で対応する View を描画 | 一旦そのまま残してよい。導線を消せば通常は表示されない。必要なら「初期公開時はこの view は表示しない」コメントを追加 |

※ シミュレーター・車両相談・開業相談・takesan-log の **View コンポーネント本体は削除せず残す**。表示されないようにするのは「トップ／お客様／ドライバー」からの導線のみでよい。

## 3. 残すべき要素

以下はそのまま残し、必要に応じて**文言・構成・デザイン**を設計書に合わせて修正する。

### 3.1 ヘッダー

- ロゴエリアの「軽貨物TAKE」テキスト・「Matching Portal」サブ（文言は設計に合わせて変更可）
- 「山口県」バッジ（エリア表記として設計と整合）
- ナビ：トップ／お客様向け／ドライバー向け（3リンク）

※ 個人メール・顔写真・LINE QR は上記「削除」のとおり削除。

### 3.2 トップページ（TopView）

- ヒーローエリアの**枠**（背景・レイアウト）は残す。**文言とボタン**を設計書に合わせて差し替え：
  - 大見出し → 「配送の相談も、ドライバー登録も。下関発の軽貨物ネットワーク。」
  - サブ見出し → 設計書の約95文字
  - メインボタン2つ：1. 配送の相談をする（→ customer または consult-delivery）、2. 協力ドライバー登録（→ driver または register）
  - バッジ・サービス3カード → 設計書の「サービス5点」に差し替え
- 2カラム「お客様向けポータル」「ドライバー向けポータル」は残す。文言を設計書の訴求文に合わせ、ボタンは「配送の相談をする」「協力ドライバー登録」に統一。
- エリア表記を「山口県下関市を中心に」「当面は下関および周辺エリア。将来的に山口県内へ広げていきます。」に変更。

### 3.3 お客様向け（CustomerTopView）

- ページ枠・「お客様向けポータル」見出しは残す。
- 「配送・お見積り依頼」のみ導線として残し、訴求文を設計書「お客様向け」「企業向け」に合わせる。企業向けは同じページ内のブロックまたはセクションで表現してよい。

### 3.4 ドライバー向け（DriverTopView）

- ページ枠・「ドライバー向けポータル」見出しは残す。
- 「協力ドライバー登録」のみ導線として残す。訴求文を設計書「ドライバー向け」に合わせる。

### 3.5 フォーム・機能

- **ConsultDeliveryView**（配送相談フォーム）：残す。送信完了メッセージの「たけさん」のみ文言修正。
- **RegisterView**（ドライバー登録フォーム）：残す。送信完了メッセージの「たけさん」のみ文言修正。基準書「フォーム送信→スプレッドシート→自動返信メール」に合わせ、LINEを「最初の入口」にしない表現に寄せる（送信完了後の案内は「受付番号を控えてください。折り返しご連絡します」などメール中心にしてもよい）。

### 3.6 フッター

- 枠は残す。文言を設計書に合わせる：
  - 「軽貨物TAKE | 山口県下関市を中心に」
  - 「配送相談・協力ドライバー登録の窓口」
  - 「プライバシー・個人情報の取扱い」
  - 電話誘導の注意は残してよい。「お問い合わせは〜」は個人メールを出さない形に（例：「お問い合わせはフォームから」）。

### 3.7 定数・GAS・データ

- `GAS_URL`、`KEI_MAKERS`、`BANKS_DATA` などフォーム用データは残す。
- `PROFILE_IMAGE_URL` はヘッダーロゴで使わなくするが、他で参照していなければ定数自体は残してよい（将来の運営者ページ用など）。
- `index.css` は現状のままでよい（Tailwind＋キーフレームのみで設計と衝突なし）。

## 4. 先に直す優先順位

設計書・修正依頼の「個人情報を出さない」「2導線を明確に」「初期公開範囲に絞る」「LINEを最初の入口にしない」に合わせ、次の順で進めるとよい。

| 優先度 | 内容 | 主な作業 |
|--------|------|----------|
| **1** | 個人情報・運営者露出を消す | ヘッダーから個人メール・顔写真・LINE QR を削除。トップから「登録ドライバー紹介」「運営者プロフィール」セクション＋「運営者プロフィールを表示」ボタンを削除。送信完了の「たけさんへ送信されました」を「軽貨物TAKEで受け付けました」等に変更。SimulatorView 内のたけさん写真・文言を削除または汎用文言に変更（シミュレーター導線を非表示にするなら優先度は下げてよい）。 |
| **2** | トップの主役を「配送相談」「協力ドライバー登録」にし、ボタン順・文言を設計書に合わせる | ヒーロー：大見出し・サブ見出し・メインボタン2つ（1. 配送の相談をする、2. 協力ドライバー登録）に変更。サービス3枚→設計書のサービス5点に変更。エリア表記を設計書どおりに。2カラムの訴求文・ボタン文言を設計書に合わせる。 |
| **3** | 初期公開範囲外の導線を非表示にする | CustomerTopView：シミュレーター・車両トラブル相談のボタンを削除または非表示。DriverTopView：開業トータル相談のボタンを削除または非表示。 |
| **4** | LINEを最初の入口にしない | フロート「LINEで相談する」ボタンを削除（上記1でヘッダーLINEは削除済み）。フォーム送信完了画面で、LINEを必須ステップにしない（受付番号の案内＋「折り返しご連絡します」を主にし、LINEは任意・将来用に回す）。 |
| **5** | エリア・フッター・各訴求文の文言を設計書に統一 | フッター文言。お客様・企業・ドライバー各ブロックの見出し・本文を設計書「公開用テキスト完成版」に合わせる。 |
| **6** | トップの構成順を設計書に合わせる（必要なら） | 設計書17：ヒーロー→サービス5点→エリア→お客様→企業→ドライバー→フッター。現状はヒーロー→2カラム→（削除後はなし）→フッターになるので、サービス5点・エリア・お客様・企業・ドライバーのブロック順を揃える。 |

- 設計（白ベース・青系・清潔感）と衝突する記述はなく、**変更不要**。

## まとめ

- **削除**：ヘッダー個人メール・ヘッダー顔写真・ヘッダーLINE QR、トップ「登録ドライバー紹介」・「運営者プロフィール」とその導線、フロートLINE、送信完了の個人名表記（文言変更含む）。
- **非表示**：お客様向けのシミュレーター・車両トラブル相談ボタン、ドライバー向けの開業相談ボタン（View 本体は残す）。
- **残す**：トップ・お客様・ドライバー・配送相談フォーム・ドライバー登録フォームの枠と機能、ヘッダー（ロゴ・ナビ）、フッター。文言・構成・ボタン順を設計書に合わせて修正。
- **優先順位**：1. 個人情報・運営者露出の削除 → 2. トップの主役・文言・ボタン順 → 3. 初期公開外の導線非表示 → 4. LINEを最初の入口にしない → 5. エリア・フッター・訴求文の統一 → 6. トップ構成順の調整。
